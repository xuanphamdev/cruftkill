### CI workflow sketch

```yaml
name: ci
on: [push, pull_request]
jobs:
  test:
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cargo test --all-features
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with: { components: rustfmt, clippy }
      - run: cargo fmt --all -- --check
      - run: cargo clippy --all-targets --all-features -- -D warnings
```

## Risks

| Risk | Mitigation |
|---|---|
| Windows-only test failures (path semantics) | dedicated `#[cfg(windows)]` tests for AppData paths; skip Unix-only tests on Windows |
| Coverage tool flakes | use stable `cargo-llvm-cov` not nightly grcov |
| TUI not testable in CI | mark as `#[ignore]`, run manually; cover behavior via unit tests on AppState reducer |
